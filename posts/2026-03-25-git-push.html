納品前夜にgit pushしたら、本番環境が真っ白になった話

2026年3月25日
失敗学び

ターミナルに表示された文字列を見て、指が凍った。

「remote: Deploying to production...」

深夜2時14分。薄暗いオフィスには自分しかいない。デスクの端で、コンビニで買った3杯目の缶コーヒーが冷めきっている。蛍光灯は半分消えていて、モニターの青白い光だけが顔を照らしていた。

納品前夜の「あとちょっと」

案件は中規模のコーポレートサイトリニューアル。制作期間3ヶ月、ページ数42ページ、総額280万円のプロジェクトだった。明日の午前10時にクライアントへ最終納品。その前に、どうしても気になるCSSのズレを1箇所だけ直したかった。

ヘッダーのロゴの余白が、デザインカンプより2px大きい。正直、誰も気づかないレベルだ。でも気づいてしまった自分が許せなかった。

修正自体は30秒で終わった。margin-topを18pxから16pxに変えるだけ。ローカルで確認。完璧。あとはpushして、自動デプロイが走って、終わり。明日の朝、胸を張って納品できる。

「あとはpushするだけ」。僕はそう呟いて、Enterキーを叩いた。

間違えたブランチ

git push origin main。

叩いた瞬間、ターミナルの出力がいつもと違うことに気づいた。差分が多すぎる。2pxの修正なのに、変更ファイルが47個。頭が真っ白になるまでに、たぶん3秒かかった。

ブランチを確認した。feature/redesign-v2。来月リリース予定の大規模改修ブランチ。テストは半分しか書いていない。APIの接続先もステージング環境のまま。それをmainにpushした。しかもCI/CDパイプラインが自動で動いている。

「嘘だろ」

声が出ていた。誰もいないオフィスに、自分の声が反響した。

28分間の恐怖

慌ててブラウザを開いた。クライアントのサイトのURLを打ち込む。指が震えていて、URLを2回打ち間違えた。

真っ白。

何も表示されない。404エラーですらない。HTMLの構文エラーでブラウザが何も描画できていなかった。ただの、白い画面。その白さが、自分の顔から血の気が引いていくのと同じ速度で広がっていった。

git revert。まずこれだ。冷静になれ。深呼吸した。キーボードに手を置く。でも指が言うことを聞かない。「git revert」を「git rebert」と打った。もう一回。「git revret」。3回目でやっと正しく入力できた。

普段なら10秒で終わるコマンドに5分かかった。revertのコミットメッセージを書く画面で、vimの終了方法を一瞬忘れた。3年間毎日使っているvimの、:wqを。

push。デプロイ待ち。CIのログを凝視する。1分が10分に感じた。額から汗が流れて、キーボードの上にポタリと落ちた。

深夜2時42分。サイトが戻った。トップページ、会社概要、サービスページ、採用情報。全ページを手動で確認した。28分間の地獄だった。

深夜3時のアクセスログ

「大丈夫、深夜にサイトを見る人なんていない」。そう自分に言い聞かせて、その夜は帰宅した。布団に入ったのは午前3時半。眠れなかった。天井を見つめながら、何度もスマホでサイトの表示を確認した。

翌朝9時15分。出社して席に着く前に、上司の田中さんに声をかけられた。

「昨日の夜、クライアントからメール来てたの見た？」

胃が縮んだ。急いでメールを開く。午前2時31分、クライアントの担当者・佐々木さんからのメール。「サイトが表示されないのですが、メンテナンス中でしょうか？」。深夜にサイトを見る人がいた。しかもクライアント本人だった。

翌朝の電話

午前10時の納品ミーティング。クライアントのオフィスに向かう電車の中で、何を言うか必死に考えた。正直に言うか。サーバーの一時的な障害とごまかすか。

会議室に入った瞬間、佐々木さんが口を開いた。

「昨夜の件、大丈夫でしたか？」

「申し訳ありません。作業中のコードを誤って本番環境に反映してしまいました。28分間、サイトが表示されない状態になっていました」

正直に言った。言い訳はしなかった。佐々木さんは3秒ほど黙ってから、「再発防止策はありますか？」と聞いてきた。

帰社後、上司の田中さんに報告した。怒鳴られると覚悟していた。でも田中さんは静かにこう言っただけだった。

「で、もう同じことは起きないんだね？」

怒鳴られるより、ずっと堪えた。

二度と繰り返さないために

その日の午後、僕は3時間かけて再発防止策を実行した。mainブランチへの直接pushを禁止するブランチ保護ルールの設定。PRのレビュー必須化。マージ前のCI通過を必須にする設定。全部、本来なら最初からやるべきことだった。

先輩エンジニアの山本さんに相談したら、「俺も2年前に同じことやったよ」と笑われた。「深夜テンションでmainに直pushして泣いたことないエンジニア、いないから」と。慰めにはなったけど、あの白い画面は忘れられない。

「あとはpushするだけ」。この言葉を聞くたびに、今でもあの午前2時14分の画面がフラッシュバックする。指先が冷たくなる。

今のチームでは、深夜のデプロイを全面禁止にしている。疲れた脳で触っていいのは個人プロジェクトだけ。本番環境は朝の、頭がクリアで、コーヒーの1杯目がまだ温かいうちに。もし今、あなたが深夜2時にターミナルを開いているなら、頼むからブランチ名をもう一度確認してほしい。
